업데이트, 플러그인 변경과 운영체제 재설치 전에는 반드시 복구 가능한 백업을 만들어야 합니다. 같은 인스턴스의 다른 폴더에만 저장한 파일은 인스턴스나 저장 장치에 문제가 생기면 함께 사라질 수 있습니다.

## 백업할 항목

가장 안전한 방법은 서버 폴더 전체를 백업하는 것입니다. 용량을 줄여야 한다면 최소한 다음 항목을 포함하세요.

* `level-name`으로 지정한 월드 폴더와 Nether·End 월드 폴더
* `plugins` 폴더의 플러그인 파일, 설정과 데이터
* `server.properties`, `bukkit.yml`, `spigot.yml`과 Paper의 `config` 폴더
* `ops.json`, `whitelist.json`, `banned-players.json`, `banned-ips.json`
* 시작 스크립트와 직접 수정한 기타 설정 파일

`logs`와 다시 내려받을 수 있는 캐시 파일은 보통 제외할 수 있습니다. 장애 분석 중이라면 관련 로그는 별도로 보관하세요.

## 안전하게 백업하기

1. 플레이어에게 점검 시간을 알립니다.
2. 서버 콘솔에서 `stop`을 입력합니다.
3. 저장 완료 메시지가 나타나고 Java 프로세스가 완전히 종료될 때까지 기다립니다.
4. 서버 폴더를 압축하거나 별도 저장 위치로 복사합니다.
5. 백업 파일의 날짜, Minecraft 버전과 Paper 빌드를 기록합니다.
6. 백업을 다른 저장 장치나 신뢰할 수 있는 외부 저장소에도 복사합니다.

<alert type="warning">
서버가 월드 파일을 쓰는 동안 폴더를 복사하면 서로 다른 시점의 파일이 섞일 수 있습니다. 별도의 온라인 백업 도구를 검증해 사용하지 않는 한 서버를 완전히 중지한 뒤 백업하세요.
</alert>
