Stella IT Console의 인스턴스 생성 화면에는 현재 사용할 수 있는 운영체제 이미지가 표시됩니다. 같은 운영체제라도 제공 버전은 바뀔 수 있으므로 문서에 적힌 과거 목록보다 Console의 선택 화면을 기준으로 결정하세요.

## Linux를 선택하기 좋은 경우

* 웹 서버, 데이터베이스, 봇처럼 화면 없이 계속 실행하는 서비스를 운영할 때
* SSH와 명령줄 사용에 익숙할 때
* 운영체제가 사용하는 메모리를 줄이고 싶을 때

처음 사용하는 경우에는 자료가 많고 패키지 설치가 쉬운 Ubuntu 계열을 선택하는 편이 무난합니다. 애플리케이션이 특정 배포판이나 버전을 요구한다면 해당 요구 사항을 우선하세요.

## Windows Server를 선택하기 좋은 경우

* Windows에서만 실행되는 프로그램이 필요할 때
* 원격 데스크톱(RDP)과 그래픽 화면이 필요할 때
* 운영하려는 프로그램이 Windows 환경을 명시적으로 지원할 때

Windows Server는 그래픽 환경과 백그라운드 서비스가 사용하는 자원이 있으므로, 같은 작업이라도 Linux보다 여유 있는 메모리가 필요할 수 있습니다.

## 선택하기 전에 확인할 사항

1. 실행할 프로그램이 지원하는 운영체제와 버전을 확인합니다.
2. 필요한 CPU, 메모리와 저장 공간을 확인합니다.
3. 관리 방법을 결정합니다. Linux는 주로 SSH를, Windows는 주로 RDP를 사용합니다.
4. 장기간 운영할 서버라면 보안 업데이트가 제공되는 버전을 선택합니다.

<alert type="warning">
운영체제를 다시 설치하거나 다른 이미지로 변경하면 인스턴스의 기존 데이터가 삭제될 수 있습니다. 변경 전에 필요한 파일을 인스턴스 외부에 백업하고 복구할 수 있는지 확인하세요.
</alert>
